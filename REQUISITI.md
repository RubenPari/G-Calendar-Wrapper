Requisiti di Sistema: G-Cal Wrapper App1. IntroduzioneL'obiettivo del progetto è lo sviluppo di un'applicazione web (SaaS-ready) che funga da interfaccia avanzata (wrapper) per Google Calendar. L'app permetterà agli utenti di autenticarsi tramite il proprio account Google, visualizzare i propri calendari e gestire gli eventi in modo sincrono.2. Stack TecnologicoBackend: FastAPI (Python 3.12+)Frontend: Angular (Versione 20+) utilizzando segnali (Signals) e Standalone Components con primeng come libreria UI.Database (opzionale per caching): PostgreSQL o Redis (per sessioni/token).Integrazione: Google Calendar REST API v3.Autenticazione: Google OAuth 2.0.3. Requisiti Funzionali3.1 Autenticazione e AutorizzazioneLogin con Google: Implementazione del flusso OAuth2 (Code Grant Flow).Gestione Scopes: Richiesta permessi per https://www.googleapis.com/auth/calendar.events e https://www.googleapis.com/auth/calendar.readonly.Gestione Token: Refresh token memorizzati in modo sicuro per mantenere la sessione attiva.3.2 Gestione Calendari ed EventiVisualizzazione: Dashboard con vista mensile, settimanale e giornaliera (utilizzando librerie come FullCalendar o componenti custom Angular).Sincronizzazione Real-time: Recupero degli eventi esistenti da Google Calendar all'accesso.CRUD Eventi:Creazione: Titolo, data/ora inizio e fine, descrizione, partecipanti, luogo.Modifica: Aggiornamento dei dettagli con sincronizzazione immediata sul server Google.Eliminazione: Rimozione dell'evento.Notifiche: Integrazione opzionale per promemoria basati sui dati di Google.3.3 Dashboard UtenteSelezione di più calendari (es. Personale, Lavoro, Festività).Filtro eventi per categoria o colore.Tema Dark/Light (supporto nativo Angular/Tailwind).4. Requisiti Tecnici & Architettura4.1 Backend (FastAPI)Endpoint Principali:GET /auth/login: Reindirizzamento all'URL di autorizzazione Google.GET /auth/callback: Gestione del ritorno da Google e scambio del codice con il token.GET /api/v1/events: Recupero della lista eventi filtrata per data.POST /api/v1/events: Creazione nuovo evento.PATCH /api/v1/events/{id}: Aggiornamento evento.DELETE /api/v1/events/{id}: Cancellazione evento.Integrazione Google SDK: Utilizzo di google-auth e google-api-python-client.4.2 Frontend (Angular 20+)Architettura: Utilizzo di @defer per caricamento pigro dei componenti pesanti (es. il calendario).State Management: Utilizzo di Signals per gestire la lista degli eventi e lo stato dell'autenticazione.Services:AuthService: Gestione token e stato login.CalendarService: Chiamate HTTP verso il backend FastAPI.Interfaccia: Tailwind CSS per uno stile moderno e reattivo.5. Requisiti Non FunzionaliSicurezza: Utilizzo di HTTPS, crittografia dei token nel database (se persistiti), e protezione CSRF.Scalabilità: Il backend deve essere stateless per supportare il deploy in container (Docker).Performance: Tempo di risposta delle API < 200ms (escludendo la latenza di Google).Usabilità: Interfaccia mobile-first.6.
